# PAIN LOG — Nhật ký pain point

> Ghi lại các khó khăn khi dùng Second Brain. Cuối tuần review để cải thiện.
> `brain-eod` sẽ append vào file này khi user cung cấp pain point.

## Định dạng mỗi entry

```
### YYYY-MM-DD
- **Pain:** mô tả ngắn
- **Context:** đang làm gì thì gặp
- **Workaround:** tạm thời làm sao để vượt qua
- **Recovery idea:** ý tưởng để fix (nếu có)
```
